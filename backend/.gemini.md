# Инструкции и требования для Бэкенда (Backend)

Этот файл содержит правила, требования и модель данных для серверной части проекта «Малая Родина». ИИ-ассистент должен руководствоваться этими правилами при работе в директории `backend/`.

---

## 1. Технологический стек бэкенда

* **Язык программирования:** Python 3.10+
* **Веб-фреймворк:** FastAPI (асинхронный режим)
* **СУБД:** PostgreSQL с расширением **PostGIS** (для хранения геоданных)
* **ORM:** SQLAlchemy (2.0 style, асинхронный сессионный режим с драйвером `asyncpg`)
* **Миграции БД:** Alembic
* **Валидация и схемы:** Pydantic v2 и `pydantic-settings` (управление конфигурацией и переменными окружения)
* **Авторизация:** JWT (JSON Web Tokens) c использованием `PyJWT` и библиотеки `bcrypt` напрямую (без `passlib`) для хеширования паролей.
* **Хранилище файлов:** S3-совместимое объектное хранилище (MinIO для локальной разработки / Yandex Object Storage) с использованием официальной библиотеки `boto3`.
* **Обработка медиа:** Pillow (для сжатия изображений), Python-multipart (для обработки форм)

---

## 2. Модели данных SQLAlchemy и Схемы Pydantic

Бэкенд оперирует следующими сущностями. Ниже приведены описания полей и типы данных для моделей SQLAlchemy.

### User (Пользователь)
* `id`: UUID (Primary Key, по умолчанию `uuid.uuid4`)
* `email`: String (Unique, Indexed, валидация через Pydantic `EmailStr`)
* `password_hash`: String
* `full_name`: String
* `avatar_url`: String (nullable, ссылка на S3)
* `created_at`: DateTime (по умолчанию UTC-время)

### Location (Локация с географической иерархией)
* `id`: UUID (Primary Key, по умолчанию `uuid.uuid4`)
* `type`: Enum (`settlement` (город/село/деревня), `street` (улица), `building` (дом/здание), `room` (квартира/комната))
* `name`: String (название населенного пункта, улицы или номер дома)
* `coordinates`: Geometry (Point, SRID 4326, nullable) — *тип данных PostGIS*. Обязателен только для типов `settlement` и `building`.
* `parent_id`: UUID (Foreign Key на `Location.id`, nullable, самореферентная связь `relationship("Location", remote_side=[id])`)
* `created_by`: UUID (Foreign Key на `User.id`)
* `created_at`: DateTime (по умолчанию UTC-время)

### Memory (Воспоминание / Пост)
* `id`: UUID (Primary Key)
* `user_id`: UUID (Foreign Key на `User.id`)
* `location_id`: UUID (Foreign Key на `Location.id`)
* `title`: String
* `text_content`: String (поддержка текста воспоминания)
* `privacy`: Enum (`public` (публичное), `private` (приватное), `family` (семейный круг))
* `family_token`: String (Unique, Indexed, nullable — генерируется при выборе приватности `family`)
* `event_date`: Date (nullable, дата события. Может храниться как неполная дата, либо отдельное строковое поле/год для гибкости)
* `created_at`: DateTime
* `updated_at`: DateTime

### MediaFile (Медиафайл)
* `id`: UUID (Primary Key)
* `memory_id`: UUID (Foreign Key на `Memory.id`)
* `uploaded_by`: UUID (Foreign Key на `User.id`) — кто именно загрузил файл (для соответствия ТЗ)
* `file_url`: String (ссылка на S3)
* `file_type`: Enum (`photo` | `video`)
* `size_bytes`: Integer
* `duration_seconds`: Integer (nullable, только для видео)
* `original_date`: DateTime (nullable, дата съёмки из метаданных EXIF)
* `created_at`: DateTime

### Resident (Житель локации)
* `id`: UUID (Primary Key)
* `location_id`: UUID (Foreign Key на `Location.id`)
* `user_id`: UUID (Foreign Key на `User.id`, nullable — если житель не зарегистрирован)
* `full_name`: String (ФИО жителя)
* `period_start`: String (например, "1980")
* `period_end`: String (например, "1989" или "по н.в.")
* `role`: String (описание связи, например, "друг детства", "бабушка", "сосед")
* `created_at`: DateTime

### PhotoTag (Отметка жителя на медиафайле)
* `id`: UUID (Primary Key)
* `media_file_id`: UUID (Foreign Key на `MediaFile.id`)
* `resident_id`: UUID (Foreign Key на `Resident.id`)
* `x_percent`: Float (0.0 to 100.0)
* `y_percent`: Float (0.0 to 100.0)
* `created_at`: DateTime

## 3. Бизнес-логика и ограничения (FastAPI-правила)

1. **Загрузка и валидация медиа:**
   * **Изображения:** Обязательное сжатие на бэкенде с помощью библиотеки `Pillow`. Изображения должны сохраняться в формате WebP со средним качеством (например, 80%) для оптимизации дискового пространства. Очистка EXIF-данных перед сохранением.
   * **Видео:** При загрузке видео проверяется размер (макс. 100 МБ) и длительность (макс. 60 секунд). Рекомендуется использовать метаданные из контейнера видео (например, библиотеки `python-multipart` и парсер заголовков) для предварительной валидации.
2. **Семейный доступ и приватность:**
   * Для постов с приватностью `family` генерируется уникальный UUID-токен (`family_token`).
   * Доступ к семейному посту предоставляется либо авторизованному автору, либо по эндпоинту с валидным query-параметром `?token=<family_token>`.
3. **Логика «Мои места» (Добавление новой локации):**
   * При создании пользователем новой локации (`Location` типа `settlement` или `building`), бэкенд должен автоматически создать запись в таблице `Resident`, связывающую этого пользователя (`User.id`) с созданной `Location.id`. При этом указывается переданный пользователем период проживания и его роль (например, "Житель" или "Владелец").
4. **Асинхронность в FastAPI:**
   * Все эндпоинты, работающие с БД, должны использовать `async def` и `async_session` от SQLAlchemy.
   * Поскольку для S3/MinIO используется синхронная библиотека `boto3`, операции ввода-вывода с хранилищем должны выполняться без блокирования главного потока: либо через объявление эндпоинтов как обычных функций `def` (которые FastAPI запускает в пуле потоков), либо с использованием `anyio.to_thread.run_sync` внутри асинхронных функций `async def`.
5. **Формат ответов API (REST):**
   * Все эндпоинты должны возвращать данные напрямую в формате JSON без использования общих конвертов (типа `{success: true, data: ...}`).
   * Для успешных запросов используются HTTP-статусы `200 OK` или `201 Created`.
   * При ошибках возвращаются соответствующие HTTP-статусы (400, 401, 403, 404, 500) со стандартным телом ошибки FastAPI: `{"detail": "Сообщение об ошибке"}`.
   * Все ответы должны быть задекорированы через `response_model` с использованием Pydantic-схем для автогенерации Swagger-документации.
6. **Обработка ошибок (Business Exceptions):**
   * Вся логика приложения при возникновении ошибок должна использовать узконаправленные, конкретные исключения, унаследованные от базовых классов (например, `NotFoundException`, `AlreadyExistsException`, `PermissionDeniedException`, `ValidationException`, которые наследуются от общего `BusinessException`).
   * Вместо выбрасывания общего исключения с текстовым параметром (например, `EntityNotFoundException("User not found")`), создаются и выбрасываются конкретные самоописывающие классы ошибок с предопределенным текстом по умолчанию:
     * `UserNotFoundException` -> "Пользователь не найден" (HTTP 404 Not Found)
     * `LocationNotFoundException` -> "Локация не найдена" (HTTP 404 Not Found)
     * `MemoryNotFoundException` -> "Воспоминание не найдено" (HTTP 404 Not Found)
     * `EmailAlreadyExistsException` -> "Пользователь с таким e-mail уже зарегистрирован" (HTTP 409 Conflict)
     * `VideoDurationExceededException` -> "Длительность видео превышает лимит 60 секунд" (HTTP 400 Bad Request)
     * `VideoSizeExceededException` -> "Размер видео превышает лимит 100 МБ" (HTTP 400 Bad Request)
     * `AccessDeniedException` -> "Доступ ограничен" (HTTP 403 Forbidden)
     * `InvalidCredentialsException` -> "Неверный e-mail или пароль" (HTTP 401 Unauthorized)
   * Исключения должны быть вынесены в **отдельный пакет `app/exceptions/`** (по аналогии со структурой пакетов в Spring Boot).
   * Каждое исключение должно находиться в **отдельном одноименном файле** в формате `snake_case` (например, класс `UserNotFoundException` находится в `user_not_found_exception.py`, а `VideoSizeExceededException` в `video_size_exceeded_exception.py`).
   * Все исключения импортируются и реэкспортируются в `app/exceptions/__init__.py` для обеспечения чистого импорта на уровне пакета.
   * В точке входа (`app/main.py`) регистрируется глобальный обработчик для базового `BusinessException`, который автоматически считывает HTTP статус-код из атрибута класса исключения и возвращает стандартный REST-ответ FastAPI: `{"detail": str(exc)}`. Это освобождает сервисный слой от работы с HTTP-кодами напрямую и делает код чистым.
7. **Настройка CORS (Cross-Origin Resource Sharing):**
   * В `app/main.py` должна быть обязательно настроена `CORSMiddleware`.
   * Список разрешенных источников (`allow_origins`) должен считываться из настроек (класс `Settings`). На локальном сервере разработки должен быть по умолчанию разрешен хост фронтенда (например, `http://localhost:5173` или `http://localhost:3000`).
8. **Управление конфигурацией (Settings и .env):**
   * Все конфигурационные данные (строка подключения к PostgreSQL/PostGIS, ключи доступа MinIO/S3, JWT-секреты, CORS-хосты) должны быть описаны в классе `Settings(BaseSettings)` из библиотеки `pydantic-settings` (в `app/core/config.py`).
   * В приложении должен использоваться гибридный подход: Pydantic Settings считывает переменные в первую очередь из системного окружения (что критично для Docker Compose при использовании `env_file`), а при их отсутствии — из локального файла `.env` (для локального запуска).
   * Файл `.env` должен располагаться в корне всего проекта (на один уровень выше папки `backend/` и `frontend/`), чтобы он был доступен как для `docker-compose.yml`, так и для бэкенда. Пароли, приватные ключи и прочие секреты не должны находиться в исходном коде.
9. **Внедрение зависимостей (Dependency Injection):**
   * Получение сессии базы данных (`async_session`), клиента S3 (`boto3`) и данных текущего авторизованного пользователя из JWT-токена должно осуществляться через механизм `Depends` от FastAPI.
   * Это обеспечивает слабую связанность компонентов и возможность легкого тестирования (mocking) при написании тестов.
10. **Структура проекта:**
   * Рекомендуется придерживаться стандартной структуры для FastAPI приложений:
     ```text
     backend/
     ├── app/
     │   ├── api/          # Эндпоинты (routers)
     │   ├── core/         # Конфигурация (config.py), безопасность, константы
     │   ├── db/           # Сессия БД, базовая модель
     │   ├── exceptions/   # Выделенный пакет для всех бизнес-исключений (Spring Boot style)
     │   ├── models/       # Модели SQLAlchemy
     │   ├── schemas/      # Схемы Pydantic
     │   ├── services/     # Бизнес-логика (работа с S3, медиа, авторизация)
     │   └── main.py       # Точка входа FastAPI (CORS, exception handlers)
     ├── migrations/       # Файлы Alembic
     ├── alembic.ini
     ├── requirements.txt
     └── .gemini.md
     ```
